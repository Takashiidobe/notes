Thus, MOVIE and LOVE have an edit distance of 2.

So we have a row and column before us to pick from.

Let's take the first cell we haven't visited yet, (1,1), and apply some rules.

Our rules are as follows: If the character in the row matches the character in the column, like in the case of the empty string and the empty string, we can take the number to the left and above us, and set that as our cell. Why? This is because we can disregard both characters (they are the same) and consider our problem to have the same edit distance as the cell if both characters didn't exist.

Take the words "go" and "no". Their edit distance is 1, because we need to update "g" to "n" or vice-versa. We do not care about "o" in both words, because they are the same. Thus, "go" and "no"'s edit distance is the same as if I asked you "g" and "n"s edit distance.

In order to make a cell that "matches" the same as not counting, we take from the left-above cell, which is the same as if both words didn't have that character.

The second rule is if there is no match, we look for the minimum of the cell left, above, and left-above, and add one to that value (since we must make a change). We keep track of the minimum cost, and carry that onto the end.

Let's apply those rules to our table:

In our current cell (1,1), we find the minimum of left, above, and left-above. That's 0. We add 1 to that, because we're comparing "M" with "L". In this case, there is an edit distance of 1 (update "M" to "L" or vice-versa).
